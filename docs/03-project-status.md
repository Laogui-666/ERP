#### 🔴 P0 严重问题（7 项）

| # | 问题 | 文件 | 影响 | 修复方案 |
|---|---|---|---|---|
| 1 | ~~客服数据范围过滤错误~~ ✅ | `rbac.ts` | CS 角色 `customerId: user.userId` 意味着客服只能看"自己是客户"的订单——逻辑完全错误 | Order 新增 `createdBy` 字段，CS 过滤改为 `createdBy: user.userId` |
| 2 | 接单并发竞争条件 | `claim/route.ts` | 两个资料员同时接单：各自查到 PENDING_CONNECTION → 都通过校验 → 后者覆盖前者 collectorId | 接单改为原子操作 `UPDATE WHERE collectorId IS NULL AND status = ?`，检查 affectedRows |
| 3 | 接单逻辑重复且不一致 | `claim/route.ts` vs `order.service.ts` | API route 调用 `transitionOrder()`，service 层自己写事务——两套代码，service 层跳过状态机校验 | 统一使用 `transitionOrder`，删除 service 层 claim 重复实现 |
| 4 | 客户自动创建后无法登录 | `orders/route.ts` POST + `login/route.ts` | 客服创建订单时自动创建客户，`passwordHash = ''`；登录时 `bcrypt.compare(password, '')` 抛异常 | 实现首次登录流程：空 hash → 强制跳转改密页 → 改密后正常登录 |
| 5 | **Socket.io 服务器未启动** | `socket.ts` | `initSocketServer()` 存在但从未调用；`npm run dev` 不会启动 Socket.io；所有 `emitToUser/emitToCompany` 为空操作 | 创建 `server.ts` 自定义启动脚本（http.createServer + Next.js + Socket.io） |
| 6 | **事件总线是死代码** | `events.ts` | 定义了完整 EventBus，但无任何 `emit()` 调用；transition 和 API routes 都不触发事件 | 在 `transitionOrder` 事务提交后 `eventBus.emit()`；API routes 创建/更新操作同理 |
| 7 | **OSS 模块是占位实现** | `oss.ts` | `uploadFile()` 返回假 URL，文件未实际上传；M2 任务 16/18 完全依赖文件上传 | 接入阿里云 OSS SDK（`ali-oss`），实现 upload/delete/getSignedUrl |

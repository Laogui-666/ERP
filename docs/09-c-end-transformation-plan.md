## 1. 现状问题诊断

### 1.1 架构定位错误

当前项目本质上是一个 **ERP 系统 + 一个简陋的品牌页**：
- 根路径 `/` → 一个模板化的"盼达旅行品牌页"（静态 Hero + 统计 + 热门国家 + 服务网格 + CTA）
- `/portal` → 另一个"Portal 首页"（HeroBanner + StatsCounter + ToolGrid + DestinationCarousel）
- 两个首页内容高度重复，定位模糊
- ERP 管理端 `/admin/*` 和客户端 `/customer/*` 才是项目的核心功能

### 1.2 C 端体验差距

| 维度 | 当前状态 | 目标状态 |
|---|---|---|
| 品牌 | "沐海旅行" + "盼达旅行" 混用 | 统一"华夏签证" |
| 首页 | 静态模板，无产品感 | Airbnb 级别成熟产品体验 |
| 导航 | 顶部导航栏 + ERP 菜单思维 | C 端底部 Tab + 顶部搜索 |
| 工具 | 6 个工具页面是占位壳 | 有 UI 设计感的服务入口 |
| 用户旅程 | 打开就看到"免费注册" | 先被内容吸引 → 自然转化 |
| ERP 暴露 | 导航栏直接有 ERP 入口 | 完全隐藏，仅"我的"页面可见 |

### 1.3 ERP 保护要求

以下文件/目录 **绝对禁止修改**：
- `src/modules/erp/` — 全部 ERP 业务代码
- `src/shared/lib/` — 共享基础设施（prisma/auth/rbac/socket/oss/transition/events 等）
- `src/shared/ui/` — 共享 UI 组件
- `src/shared/stores/` — 共享状态管理
- `src/shared/hooks/` — 共享 Hooks
- `src/shared/types/` — 共享类型
- `src/app/admin/` — 管理端全部页面
- `src/app/customer/` — 客户端全部页面
- `src/app/api/` — 全部 API 路由
- `src/app/(auth)/` — 认证页面（仅品牌文字可改）
- `src/middleware.ts` — 中间件（仅添加路由，不改逻辑）
- `prisma/` — 数据库 Schema 和迁移
- `server.ts` — Custom Server
- 全部配置文件（tsconfig/vitest/next.config 等）

## 5. 首页重设计（Airbnb 风格）

### 5.1 设计参考

参考 Airbnb 首页设计哲学：
- **大 Hero 区**：不是"注册按钮"，而是"搜索+灵感"——让用户有事可做
- **卡片化内容**：目的地卡片、服务卡片，图片驱动而非文字驱动
- **个性化推荐**："热门目的地""精选服务""近期优惠"
- **社交证明**：用户评价、成功案例、数据统计
- **自然转化**：内容吸引 → 产生兴趣 → 尝试工具 → 注册下单

### 6.1 顶部导航栏（AppNavbar）

```
┌───────────────────────────────────────────────────────────┐
│ [🌍 华夏签证]  [🔍 搜索国家、签证类型...]        [登录] [注册] │
│                                        或                  │
│                              [🔔 通知] [👤 头像▾]          │
└───────────────────────────────────────────────────────────┘
```

- 滚动前：透明背景
- 滚动后：毛玻璃背景 + 底部阴影
- 移动端：搜索栏收起为搜索图标

### 6.2 底部 Tab（AppBottomTab）

```
┌───────────────────────────────────────────────────────────┐
│  🏠 首页    🛂 服务    🧰 工具    📋 订单    👤 我的       │
│                                    [1]                     │
└───────────────────────────────────────────────────────────┘
```

- 固定底部，safe-area 适配
- 订单 Tab 有未读角标（有待办事项时）
- active 状态：颜色 + 底部指示条

### 6.3 与现有 portal layout 的关系

当前 `src/app/portal/layout.tsx` 是 4 Tab（首页/订单/消息/我的）。改造后：
- 底部 Tab 改为 5 Tab（首页/服务/工具/订单/我的）
- "消息"Tab 移除（通知通过铃铛图标进入，不在底部 Tab 占位）
- layout 路径从 `/portal/*` 改为顶层路由 `/`

**关键决策**：为了不破坏 ERP 的 `/admin/*` 和 `/customer/*` 路由，新的 C 端页面使用顶层路由：
- `/` → 首页
- `/services` → 服务
- `/tools/*` → 工具
- `/orders` → 我的订单
- `/profile` → 我的

Portal layout 的职责改为：包裹这些 C 端页面，提供底部 Tab + 顶栏。

**但**：现有 `/portal/*` 路由下的工具页面（已有 8 个 Model + 7 个 API）需要保留。策略：
- 新首页 `/` 直接在根路径
- `/portal` 重定向到 `/`
- `/portal/tools/*` 重定向到 `/tools/*`
- `/portal/orders` 重定向到 `/orders`
- `/portal/profile` 重定向到 `/profile`
- `/portal/notifications` 重定向到 `/profile`（通知中心移入"我的"）
